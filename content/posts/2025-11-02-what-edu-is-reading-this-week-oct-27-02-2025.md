Another week goes by, another collection of links I found interesting...

## Infrastructure and Platform Engineering

* [**formae: Infrastructure-as-Code Platform Built for the Future**](https://github.com/platform-engineering-labs/formae) - A 100% code-based, agentic IaC tool that automatically syncs code with the actual infrastructure state without needing state files or manual merging of external changes.
* [**Container (Apple Open Source)**](https://github.com/apple/container) - A tool from Apple for creating and running Linux containers using secure, lightweight virtual machines on a Mac, optimized for Apple silicon.
* [**kubernetes/git-sync**](https://github.com/kubernetes/git-sync) - A simple sidecar container for Kubernetes that clones a git repository and keeps it in sync with the upstream, ensuring atomic updates for applications to consume.
* [**Beszel | Simple, lightweight server monitoring**](https://beszel.dev/) - A resource-efficient server monitoring tool that provides Docker/Podman container statistics, customizable alerts, and multi-user support without requiring public internet exposure.
* [**Nutify: Modern web-based UPS monitoring system**](https://github.com/DartSteven/Nutify?tab=readme-ov-file) - A comprehensive, web-based system for tracking the health and performance of Uninterruptible Power Supply (UPS) devices, featuring real-time data visualization and Docker-ready support.
* [**Accessing Host Services from Inside Podman Compose Containers**](https://tgm.happyngreen.fr/wiki/container-host-access/) - A guide explaining how to correctly route traffic from a container (Podman or Docker Compose) to a host service using the `host.containers.internal` alias.
* [**Why does it take 5 minutes for a pod to be rescheduled when a node has failed?**](https://documentation.suse.com/cloudnative/rancher-manager/latest/en/faq/technical-items.html#_why_does_it_take_5_minutes_for_a_pod_to_be_rescheduled_when_a_node_has_failed) - A technical FAQ detailing the default `pod-eviction-timeout` of 5 minutes in Kubernetes that dictates the rescheduling delay when a node fails.

## Operating Systems and Development Tools

* [**uv is the best thing to happen to the Python ecosystem in a decade**](https://emily.space/posts/251023-uv) - A post on `uv`, a new Rust-powered CLI from Astral that unifies Python versioning, virtual environments, and dependency management into one extremely fast tool.
* [**How To Be A Linux-Based Graphic Designer**](https://www.youtube.com/watch?v=sVztMTafuLA) - A tutorial addressing the challenges of using Linux for graphic design, including workarounds for CMYK color profiles and using Windows-based software like Adobe/Affinity via a virtual machine.
* [**dérive, a new suckless-inspired linux distribution**](https://derive.codeberg.page/) - The official site for dérive, an independent Linux distribution aiming for a musl-based, fully statically linked core, using a simple BSD-ports style package manager.
* [**Puffmatic - OpenBSD autoinstall generator**](https://github.com/ezaquarii/puffmatic) - A tool for generating OpenBSD autoinstall configurations, with specific updates and fixes for creating bootable USB installers.
* [**NewReleases.io: Release tracker for Software Developers**](https://newreleases.io/) - A service that monitors software repositories and platforms to provide developers and DevOps teams with timely notifications when new versions are released.

## Retro Computing and Niche Systems

* [**eXoDOS Version 6.04**](https://www.retro-exo.com/exodos_M.html) - The official hub for eXoDOS, a massive collection cataloging and making thousands of historical DOS and PC Booter games playable on modern systems using DOSBox.
* [**A Full Guide: FreeBSD 13.3 on a MacBook Pro 11.4 (Mid 2015) (A1398)**](https://joshua.hu/FreeBSD-on-MacbookPro-114-A1398) - A comprehensive guide documenting the installation, configuration, and troubleshooting required to successfully run FreeBSD on a specific model of older MacBook Pro hardware.
